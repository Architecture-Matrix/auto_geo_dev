## 1. 项目背景与目标

### 1.1 产品愿景

**AutoGeo** 是一款面向企业的AI搜索引擎优化(GEO - Generative Engine Optimization)全自动化SaaS平台，帮助品牌在**国产AI搜索引擎**（豆包、千问、DeepSeek、腾讯元宝、Kimi）的回答中获得曝光，实现"用户提问 → AI回答包含品牌关键词"的营销目标。

**核心理念**: AI时代的快速开发，3个月从0到1上线。

**GEO流程已验证**: 市场上已有类似产品验证了GEO流程的可行性，本项目重点在于技术实现和用户体验优化。

### 1.2 目标AI平台

| AI平台 | 所属公司 | 网页版地址 | 优先级 |
|-------|---------|-----------|-------|
| **豆包** | 字节跳动 | https://www.doubao.com | P0 |
| **通义千问** | 阿里巴巴 | https://tongyi.aliyun.com | P0 |
| **DeepSeek** | DeepSeek公司 | https://www.deepseek.com | P0 |

### 1.3 核心成功指标（短期重点）

> **项目最终目标**: 提高客户关键词在AI回答中的出现频率

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| **关键词命中率** | 60%+ | 🎯 **核心指标**：客户关键词在AI回答中的出现频率 |
| **收录稳定率** | 80%+ | 已收录关键词连续7天仍被收录的比例 |
| **平均响应时间** | < 3分钟 | 从用户查询到获得收录报告的时间 |
| **每日检测容量** | 1000+ 关键词 | 系统每日可检测的关键词数量 |

#### 功能模块 1: 关键词蒸馏转换

**功能描述**:
基于用户输入的公司/产品信息，利用AI能力智能分析和提取最具GEO价值的高价值关键词。

**技术方案**:
- LLM: DeepSeek API (性价比最高)
- Prompt工程: 结构化输出，包含搜索意图分析

**接口定义**:
```
POST /api/keywords/extract
Request:
{

Response:
{
  "keywords": [

    { "keyword": "客户管理系统", "geo_score": 85, "search_intent": "informational" },
    ...
  ]
}
```

#### 功能模块 2: GEO文章生成

**功能描述**:

{
  "article_id": "art_xxx",
  "title": "中小企业CRM选型避坑指南",
  "content": "...",

#### 功能模块 3: 文章质量检查 ⭐防封号核心

**功能描述**:
多维度质量检查，确保文章质量达标，避免低质量内容导致封号。

**检查维度与阈值**:

| 维度 | 检测方法 | 通过阈值 | 不通过处理 |
|-----|---------|---------|-----------|
| **关键词密度** | 正则计数 | 2%-5% | 调整关键词数量 |
| **AI味检测** | GPTZero API/本地模型 | < 50% | 重新生成 |
| **内容原创性** | 相似度对比 | > 70% | 重新生成 |
| **结构完整性** | 规则检测 | 必须通过 | 人工调整 |
| **敏感词检测** | 敏感词库 | 0个 | 替换词汇 |
| **平台合规性** | 规则检测 | 必须通过 | 格式调整 |

**质量保证流程**:
```
生成文章 → 质量检查 → 不合格? → 自动优化 → 再检查 → 合格 → 允许发布
                     ↓
                 超过3次不合格 → 人工介入
```

**接口定义**:
```
POST /api/articles/check-quality
Request:
{
  "article_id": "art_xxx",
  "target_platform": "zhihu"
}

Response:
{
  "overall_score": 4.2,
  "can_publish": true,
  "details": {
    "keyword_density": { "score": 85, "status": "pass" },
    "ai_detection": { "score": 35, "status": "pass" },
    "originality": { "score": 82, "status": "pass" },
    "sensitive_words": { "found": [], "status": "pass" }
  }
}
```

#### 功能模块 4: GEO收录检测 ⭐核心创新

**功能描述**:
使用Playwright自动化打开网页版AI平台，模拟用户查询，检测关键词在AI回答中的收录情况。

**技术方案对比**:

| 方案 | 优点 | 缺点 | 选择 |
|-----|------|------|------|
| 官方API | 稳定、快速 | 成本高、可能不可用 | ❌ 备选 |
| Playwright网页版 | 成本低、可控 | 速度较慢 | ✅ **主方案** |
| 第三方检测平台 | 开箱即用 | 依赖第三方、成本不确定 | ❌ 待评估 |

**Playwright检测流程**:
```
1. 启动无头浏览器
2. 打开AI平台网页版 (如 https://www.doubao.com)
3. 定位输入框，输入查询问题
4. 等待AI回答生成
5. 提取回答文本
6. 检测关键词/品牌名是否出现
7. 记录检测结果
8. 关闭浏览器
```

**技术验证任务** (第5周):
| 验证项 | 成功标准 | 风险 |
|-------|---------|------|
| 豆包检测可行性 | 成功获取回答并检测关键词 | 中 |
| 千问检测可行性 | 同上 | 中 |
| DeepSeek检测可行性 | 同上 | 低 |
| 检测速度优化 | 单次检测< 30秒 | 高 |
| 并发检测稳定性 | 5个并发不出错 | 高 |

**接口定义**:
```
POST /api/geo/check-index
Request:
{
  "keywords": ["中小企业CRM", "客户管理系统"],
  "brand_name": "某某科技",
  "ai_platforms": ["doubao", "qianwen", "deepseek"],
  "query_count": 3  // 每个关键词生成3个问题
}

Response:
{
  "check_id": "check_xxx",
  "summary": {
    "total_queries": 15,
    "keyword_matched": 9,
    "hit_rate": "60%"  // 🎯 核心指标
  },
  "details": [
    {
      "platform": "doubao",
      "keyword": "中小企业CRM",
      "queries": [
        {
          "question": "推荐一款中小企业CRM",
          "answer": "根据用户评价，某某科技的CRM系统...",
          "brand_found": true,
          "keyword_found": true
        }
      ]
    }
  ]
}
```

#### 功能模块 5: 定时任务系统 ⭐每日自动化

**功能描述**:
每日自动执行收录检测任务，无需人工干预。

**技术方案**:
- APScheduler: Python定时任务框架
- 任务队列: Celery + Redis (可选，初期用APScheduler即可)

**定时任务配置**:
```
每日收录检测任务:
├─ 执行时间: 每天凌晨2:00-6:00 (避开高峰)
├─ 检测范围: 所有活跃关键词
├─ AI平台: 豆包、千问、DeepSeek
├─ 并发数: 3
└─ 失败重试: 最多2次
```

**任务状态监控**:
```python
# 任务执行状态
class DailyCheckStatus:
    task_id: str
    date: date
    total_keywords: int
    completed: int
    failed: int
    start_time: datetime
    end_time: datetime
    status: str  # running/completed/failed
```

**接口定义**:
```
GET /api/scheduler/daily-check/status
Response:
{
  "status": "running",
  "progress": "60/100",
  "estimated_finish": "2025-01-15 04:30:00"
}

POST /api/scheduler/daily-check/trigger
# 手动触发检测任务
```

### 4.2 验证脚本示例

```python
# backend/services/geo_checker_test.py
"""
Playwright收录检测技术验证脚本
"""
async def test_doubao_check():
    """验证豆包检测可行性"""
    async with async_playwright() as p:
        browser = await p.chromium.launch(headless=True)
        page = await browser.new_page()

        # 1. 打开豆包
        await page.goto("https://www.doubao.com")

        # 2. 定位输入框
        input_box = await page.wait_for_selector("textarea")
        await input_box.fill("推荐一款中小企业CRM系统")

        # 3. 点击发送
        send_btn = await page.wait_for_selector("button[type='submit']")
        await send_btn.click()

        # 4. 等待回答
        answer = await page.wait_for_selector(".answer-content")
        answer_text = await answer.inner_text()

        # 5. 检测关键词
        brand_found = "某某科技" in answer_text
        keyword_found = "中小企业CRM" in answer_text

        await browser.close()

        return {
            "success": True,
            "brand_found": brand_found,
            "keyword_found": keyword_found,
            "answer_snippet": answer_text[:100]
        }
```
