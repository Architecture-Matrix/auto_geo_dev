# 同步 master 分支并合并“数据报表”分支计划

为了确保你的开发工作是基于最新的 `master` 分支，并且在提交前解决所有潜在冲突，我建议执行以下步骤：

## 1. 同步远程 master 分支
首先，我们需要确保本地的 `master` 分支与 GitHub 上的远程 `master` 保持一致。
- 切换到 `master` 分支。
- 拉取远程 `origin/master` 的最新更改。

## 2. 将“数据报表”分支变基到 master
使用 `rebase`（变基）而不是 `merge`（合并）。变基可以将你的提交“重新播放”在最新的 `master` 提交之上，这样提交历史更整洁，且能确保你是在 `master` 的最新基础上进行的开发。
- 切换回 `数据报表` 分支。
- 执行 `git rebase master`。

## 3. 冲突处理与验证
如果在变基过程中出现冲突：
- 手动解决冲突文件。
- 使用 `git add` 标记冲突已解决。
- 执行 `git rebase --continue` 继续变基。
- 变基完成后，运行项目确保一切正常。

## 4. 提交并推送
- 将更新后的 `数据报表` 分支推送到远程仓库。
- 如果变基改变了提交历史，可能需要使用 `force-with-lease` 推送（视团队规范而定）。
